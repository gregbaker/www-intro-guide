So far, we have only been able to select every instance of a particular tag in our stylesheets. This is often very useful, but limiting. There are many more ways to select content for modification in CSS, and we should explore some.

Classes and IDs

There are often cases where we use one element for different purposes on a page. For example, suppose we are creating a web page for a recipe: we might have lists like this:

{{ quoted_code('_sources/recipe1.html') }}

We have both ordered and unordered lists on this page, so we can easily select them separately and style them. But we have two unordered lists doing different jobs: one is a list of ingredients and one is a list of problems we think someone following the recipe might have. These are both lists where order isn't important so <ul> is the right tag to use, but it would be totally reasonable to want these to look different on the page.

In order to select these lists separately, they need to be distinguished somehow. To do this, we can use the class property. Elements can be given a class to indicate their specific role on the page. The value for a class can be any word that is meaningful. For example, we might add classes like this:
