So far, we have only been able to select every instance of a particular tag in our stylesheets. This is often very useful, but limiting. There are many more ways to select content for modification in CSS, and we should explore some.

Classes

There are often cases where we use one element for different purposes on a page. For example, suppose we are creating a web page for a recipe: we might have lists like this:

{{ quoted_code('_sources/recipe1.html') }}

We have both ordered and unordered lists on this page, so we can easily select them separately and style them. But we have two unordered lists doing different jobs: one is a list of ingredients and one is a list of problems we think someone following the recipe might have. These are both lists where order isn't important so <ul> is the right tag to use, but it would be totally reasonable to want these to look different on the page.

Actually, a class value must start with a letter, but can contain letters, number, hyphens, and underscores.

In order to select these lists separately, they need to be distinguished somehow. To do this, we can use the class property. Elements can be given a class to indicate their specific role on the page. The value for a class can be any word that is meaningful. For example, we might add classes like this:

{{ quoted_code('_sources/recipe2.html') }}

The class property won't change the page in any way by itself, but we can now uses these classes to grab particular elements in CSS:

{{ quoted_code('_sources/classes.css') }}

The second selector here (“ul.problems”) is a class selector. It selects all <ul> elements that have class="problems" but does not change other unordered lists. The first selector (“ul”) still selects all <ul> elements, whether they have a class or not.

IDs

We can also distinguish elements with the id property. The difference between class and id is that a particular id value can occur only once on a page. So, we use id to uniquely identify things.

For example, we know we will only have one page footer (information at the bottom of the page about the site, etc.). There is a tag <footer>, but we might have something like copyright information inside it that we want to style separately: we know there will only be one copyright statement on a particular page, so we use id:

{{ quoted_code('_sources/copyright.html') }}

Then we can select just the paragraph with that identifier like this:

{{ quoted_code('_sources/ids.css') }}

Above, we used class="ingredients" instead of id="ingredients" because we could create a page with multiple recipes, so multiple lists of ingredients. Here we know the copyright info will occur only once so we can use id="copyright".

Since id values are unique, they can also be used to identify a part of the page (thus the name). We will use this in the unit on {{ xref('js') }} and beyond, but for now, it can be used to link to a particular position on a page. For example, this topic starts with an element that has an id specified. We can link to that part (id="identifiers") of the page (css-selectors.html) in HTML like this:
