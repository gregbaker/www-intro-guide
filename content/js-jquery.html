Libraries in Programming

JavaScript is a full-featured programming language all by itself, but there are many tasks that programmers creating JavaScript need to do a lot which are not in the “core” language that is built into web browsers.

Every programming language is like this: the language itself lets programmers solve problems, but as the language is used, it becomes clear that some tasks must be done often (by at least some programmers).

jQuery

JQuery is a very common JavaScript library (used on >60% of the most-visited web sites). It makes several common taks easier for the programmer: manipulating the HTML page, using events, animation.

There are lots of new concepts in this JavaScript file. It probably makes sense to take a look line-by-line.

do_something = function() { … }
Just like before, this creates a function named do_something. Also like before, this function can be called with do_something() to run the code inside the { … }.
alert('Hello world!');
This one line is the code that runs when the do_something function is called. It pops up an alert window with the phrase “Hello world!”. That's maybe not too useful, but at least we can see when the function runs.

The function alert() is built into JavaScript, so we can call it without defining it.

setup = function() { … }
We are defining this function to do the work of setting up the JavaScript behaviour on the page (thus the name “setup”). It will be called after the browser loads the page (because we'll trigger that below).
jQuery('p').click(do_something);

Aw crap, what do I say here? Nothing? Everything won't fit.

jQuery(document).ready(setup);
This line gets everything started. The jQuery(document).ready(…) idiom gives a function that should be called once the browser finishes loading the page (i.e. it is “ready”). In our case, we want the setup function called when the page is ready, so we indicate it here.
